Overview

We made use of the Jarvis March Algorithm along with the KirkPatrick Seidel Algorithm to compute the Convex hull of a set of points. The project contains a graphical visualization of the working of the computation, along with a canvas to input points into.

The algorithms are written in vanilla javascript an excellent choice for web applications and is well documented with the help of Doxygen pages.

Jarvis March Algorithm

The Jarvis March algorithm, also known as the Gift Wrapping algorithm, is a computational geometry method used to find the convex hull of a set of points. The convex hull can be thought of as the smallest convex polygon that encompasses all the points in the set.

The algorithm begins by identifying the leftmost point, or the point with the smallest x-coordinate, and designates it as the current point. This is the starting point for the algorithm.

Next, the algorithm searches through all the remaining points to find the one that is most counterclockwise to the current point. This is achieved by comparing the orientation of triples of points. The point found to be the most counterclockwise is then set as the current point.

This process is repeated, with the algorithm continuously finding the most counterclockwise point from the current point, until it eventually returns to the starting point. At this stage, the algorithm has successfully computed the convex hull.

In terms of time complexity, the Jarvis March algorithm operates at O(nh), where 'n' represents the number of points and 'h' represents the number of points on the convex hull. This makes it less efficient than other convex hull algorithms, particularly when dealing with larger inputs. However, the Jarvis March algorithm can be simpler to understand and implement, making it a popular choice in certain scenarios.

KirkPatrick Seidel Algorithm

The Kirkpatrick-Seidel algorithm, also known as the ultimate planar convex hull algorithm, is a divide-and-conquer approach to compute the convex hull of a set of points in a plane. The convex hull is the smallest convex polygon that encompasses all the points.

The algorithm starts by identifying the points with the smallest and largest x-coordinates, which are guaranteed to be part of the convex hull. These points divide the remaining points into two sets: those that are above the line connecting the two points (upper hull) and those that are below (lower hull).

The algorithm then applies a "prune-and-search" technique, a variant of the median-finding algorithm. For each set of points (upper and lower hull), the algorithm finds the point that forms the maximum area triangle with the line segment as the base. This point is also part of the convex hull.

The process is then recursively applied to the points that are left of this point and above the line (for the upper hull), and to the points that are right of this point and below the line (for the lower hull). This divide-and-conquer approach continues until all points on the convex hull are identified.

The Kirkpatrick–Seidel algorithm has a worst-case time complexity of O(n log h), where 'n' is the number of points and 'h' is the number of points on the convex hull. This makes it more efficient than other convex hull algorithms such as the Gift Wrapping algorithm for larger inputs, but it is more complex to understand and implement.

Inferences

As described in the paper "On the ultimate convex hull algorithm in practice" by MM McQueen and GT Toussaint, The results suggest that although the O(n Log h) algorithms may be the ‘ultimate’ ones in theory, they are of little practical value from the point of view of running time. This is seen in our case as well where Jarvis March algorithm(O(nh)) although being very bad in theory as compared to KPS algorithm(O(nlogh)), still outperforms it practically. This is supposedly due to the large time costant associated with the KPS algorithm.

Ref: McQUEEN, M. M., & Toussaint, G. T. (1985). On the ultimate convex hull algorithm in practice. Pattern Recognition Letters, 3(1), 29-34.
